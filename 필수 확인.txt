깃허브에 있는 폴더, 파일 등 수정
  - 자신이 수정할 폴더, 파일 카톡에 명시하고 업로드가 끝나면 카톡에 완료 문구 남기기
    (ex. Doc/01.Hadoop 설치(수정) -> Doc/01.Hadoop 설치(완료))
  - Environment, Code, Data폴더의 경우에는 본인이 바로 수정 파일 업로드
    - Code의 경우 처음 작성자의 경우 코드에 작성자, 출처, 작성일자 명시
    - Code 수정일 경우 코드에 수정일자, 수정자 주석에 표시하고 기존 코드는 주석처리하고 code이름_@로 저장
      (test.py파일을 수정했을 경우 test_2.py로 업로드)
  - Doc폴더의 경우
    - 처음 올릴때는 작성자가 바로 업로드
    - 수정일시는 수정 내용을 팀장에게 수신후 팀장이 업로드

깃허브 내용중 바꾸고자 하는 것이 있을 경우 (ex. 폴더 이름, 파일 이름, 디렉터리 구성 등 깃허브에 대한 내용만)
  - 텍스트 파일을 etc폴더 밑에 이름_날짜로 만든후 내용 입력하면 다음 회의때 확인
